## Applied to One-Year Survival Prediction After Cardiac Surgery

### Overview

This repository contains the complete analysis code and a synthetic dataset for the manuscript:

> *Causal Adjacency Examination (CAE): A Formalised Leakage-Detection Procedure for Clinical Machine Learning Pipelines — Applied to One-Year Survival Prediction After Cardiac Surgery*  
> Ali Sadegh-Zadeh — submitted to *Artificial Intelligence Review* (Springer)

CAE is a graph-based algorithm that classifies each feature in a supervised learning pipeline according to its temporal and causal relationship to the outcome. The procedure assigns a risk score λ ∈ {0, 1, 2, 3} to each feature and operates on a directed acyclic graph (DAG) with a defined prediction timepoint T₀.

### Data

**Original cohort:** 113 post-cardiac surgery patients, Urmia University of Medical Sciences (UMSU), Iran. Ethics approval: IR.UMSU.REC.1403.198. The original patient-level data cannot be shared publicly; it is available from the author under a formal data-use agreement.

**Synthetic dataset** (`data/synthetic_cardiac_cae_public.csv`): A privacy-safe dataset (N = 300) generated by fitting a Gaussian copula to the marginal distributions and rank correlations of the original cohort, stratified by outcome class. It preserves statistical structure without containing any individual patient record. It is provided to allow full pipeline execution and verification of the CAE framework without requiring access to patient data. AUC on the synthetic dataset is not expected to match the original cohort.

### Method Summary

The pipeline implements:

1. **CAE classification** — each of 55 features assigned λ ∈ {0, 1, 3} based on temporal availability at T₀ = discharge and direct-cause status
2. **Corrected nested cross-validation** — 5-fold outer × 3-fold inner stratified CV; SMOTE applied exclusively inside each inner training fold
3. **12 classifiers** with inner-loop hyperparameter selection
4. **Bootstrap 95% CI** (n = 2000 stratified resamples)
5. **Multi-instrument calibration** — Brier score, ECE, Hosmer-Lemeshow χ²
6. **Progressive four-tier sensitivity analysis** — CAE=0 only → exclude contested λ=1 → all 54 features
7. **Leave-one-out marginal analysis** — ECMO, CRRT, Furosemide individually removed
8. **Furosemide ordinal encoding analysis** — ordinal {0,1,100} vs binary flag vs absent

### Key Results (original cohort)

| Model | AUC | 95% CI | HL p |
|---|---|---|---|
| SVM (Linear) | 0.952 | [0.868, 0.999] | 0.111 ✓ |
| Logistic Regression | 0.947 | [0.829, 1.000] | 0.008 ✗ |
| Gradient Boosting | 0.938 | [0.831, 1.000] | 0.517 ✓ |
| Decision Tree | 0.932 | [0.828, 1.000] | 1.000 ✓ |

**Conservative estimate (Furosemide excluded, Tier-2):** AUC = 0.810 [0.672–0.924]

### Installation

```bash
git clone https://github.com/salisadegh/cardiac-cae-ml.git
cd cardiac-cae-ml
pip install -r requirements.txt
```

### Usage

```bash
# Run full analysis on synthetic data
python analysis_main.py --data data/synthetic_cardiac_cae_public.csv

# Run sensitivity analysis
python sensitivity_analysis.py --data data/synthetic_cardiac_cae_public.csv

# Run simulation study (domain-independent CAE validation)
python simulation_study.py
```

Results are written to `results/`.
